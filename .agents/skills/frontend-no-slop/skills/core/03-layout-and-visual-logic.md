# 03 — Layout and Visual Logic

Reason in interface structure, not in vibes.

For page and screen work, specify:

- section order
- primary action vs secondary action
- dominant content block per viewport
- grouping, spacing, and alignment logic
- hierarchy of heading, support copy, proof, and controls
- responsive behavior from wide desktop down to narrow mobile

Useful lenses:

## Information Hierarchy
State what the user should notice first, second, and third.

## Spatial Logic
Name the container width, number of columns, stacking behavior, sticky regions, and overflow risks.

## Scanning Path
Explain where the eye travels and whether that path supports the user's main task.

## Density
Call out when a screen is too sparse, too dense, or hiding critical actions behind decoration.

Bad pattern:
- giant hero, vague headline, weak proof, buried CTA

Better pattern:
- sharp headline, concise subhead, immediate evidence, one primary CTA, one lower-risk secondary path
