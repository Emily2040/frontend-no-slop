# 02 — Slop Scrubber

Use `registry/forbidden-slop.json` as an explicit ban list.

Whenever you see a vague claim, convert it into something physical:

- "clean" -> spacing scale, alignment, reduced visual noise, fewer competing actions
- "modern" -> actual typography, density, motion, contrast, and layout choices
- "intuitive" -> visible affordances, obvious labels, predictable hierarchy, clear feedback
- "premium" -> material choices in the UI: spacing, restraint, imagery, proof, and copy tone

Do not output:
- generic praise
- decorative filler
- empty mood-board language
- made-up business proof
- implementation advice with no selectors, components, states, or semantics behind it

Every recommendation should answer at least one of these:
- What changes on screen?
- Why does it help the user's task?
- How would an engineer or designer apply it?
- What state, breakpoint, or accessibility consequence does it have?
