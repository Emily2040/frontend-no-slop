# Frontend Brief Template

## Surface
Describe the page, flow, or component.

## Primary User Job
What must the user accomplish here?

## Audience
Who uses this surface?

## Existing Problems
List layout, copy, state, accessibility, or performance issues.

## Constraints
Framework, brand, accessibility, responsive, or business constraints.

## Existing Assets
Code, screenshots, wireframes, tokens, or none.

## Required Output
Design critique, page plan, component spec, copy rewrite, or implementation plan.
